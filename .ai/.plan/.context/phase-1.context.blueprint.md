# PHASE 1 CONTEXT BLUEPRINT: membership-hub

## 1. Phase Operational Scope & Objectives
- **Project scaffolding** under `./sources/` with a clean, enterprise‑grade directory layout.
- **Backend framework initialization** using Quarkus 3.x (Java 17) with essential extensions (RESTEasy, Hibernate ORM, PostgreSQL JDBC, Kafka client, SmallRye OpenAPI, JWT).
- **Database schema design** for a multi‑tenant PostgreSQL model covering Centers, Users (internal & external), Roles, Courses, Enrollments, Students, Teachers, Attendance (QR), Notifications, and Audit logs. Include tenant isolation, soft‑delete, encryption of PII, and OWASP‑aligned controls.
- **Docker configuration** – a minimal Quarkus micro‑image, `docker‑compose` for local dev (Postgres + Kafka), and health‑check endpoints.
- **GCP preparation** – Cloud Build pipeline and Kubernetes manifests (Deployment, Service) ready for GKE rollout.
- **Technical documentation** – architecture diagram, OpenAPI spec, and OWASP compliance matrix stored under `./sources/backend/doc/`.

## 2. Allowed Technical Scope & Directory Boundaries (Files, paths, and endpoints)
- **Backend root:** `./sources/backend/`
  - `./sources/backend/src/main/java/org/nlh4j/saas/membershiphub/` *(Java package foundation)*
  - `./sources/backend/src/main/resources/`
  - `./sources/backend/src/main/docker/`
  - `./sources/backend/src/test/java/org/nlh4j/saas/membershiphub/`
  - `./sources/backend/docker/` *(Docker‑compose & ancillary scripts)*
  - `./sources/backend/gcp/` *(Cloud Build & K8s manifests)*
- **Frontend placeholder (future phases):** `./sources/frontend/`
- **Documentation:** `./sources/backend/doc/` *(`.md`, `.json`, `.puml` assets)*
- **REST/GraphQL endpoints** will be generated by Quarkus SmallRye OpenAPI; all paths will be prefixed with `/api/v1/*` and include a `tenant-id` header for multi‑tenant routing.

## 4. Phase Definition of Done (DoD)
- **Structural completeness:** All source files, configs, Docker & GCP artifacts exist under `./sources/` and follow the mandated path prefixes.
- **Build & run:** `mvn clean package` succeeds; `mvn quarkus:dev` starts a responsive Quarkus instance listening on `0.0.0.0:8080`.
- **Schema readiness:** PostgreSQL DDL generated (via Hibernate `create` or Flyway) creates all required tables with tenant isolation, indexes, and constraints.
- **Containerization:** `docker build -t membershiphub-backend .` produces a runnable image; `docker‑compose up` launches backend, Postgres, and Kafka without errors.
- **CI/CD readiness:** `cloudbuild.yaml` is syntactically valid and can trigger a build that pushes an image to Artifact Registry.
- **Documentation:** `architecture.md` and `api-spec.json` are present, version‑controlled, and contain the required OWASP compliance matrix and API contract.

### DAY 2: Database schema design and entity modeling
#### SUB‑TASK 2.1: Design and generate Postgres schema entities
##### Assigned Sub-Agent: coder
##### Targeted Components & Technical Requirements:
*   **Target Path:** `./sources/backend/src/main/java/org/nlh4j/saas/membershiphub/domain/Center.java`
    *   **Architectural Requirements:**
        *   `@Entity` with table `centers`; fields `id`, `name`, `address`, `taxId`, `contactPhone`, `createdAt`, `updatedAt`, `tenantId`.
        *   Apply `@TenantFilter` for multi‑tenant isolation; include `isDeleted` flag for soft delete.
        *   Encrypt `taxId` using `EncryptionService`; validate `name` length with `@Size`.
*   **Target Path:** `./sources/backend/src/main/java/org/nlh4j/saas/membershiphub/domain/User.java`
    *   **Architectural Requirements:**
        *   Entity for authentication; fields `id`, `email`, `passwordHash`, `role` (enum), `centerId`, `tenantId`.
        *   Store passwords as BCrypt hashes (OWASP A02).
        *   Add `@TenantFilter` and audit timestamps.
*   **Target Path:** `./sources/backend/src/main/java/org/nlh4j/saas/membershiphub/domain/Course.java`
    *   **Architectural Requirements:**
        *   Entity with `id`, `title`, `description`, `startDate`, `endDate`, `teacherId`, `centerId`, `tenantId`.
        *   Validate date ranges; enforce unique constraint on `(centerId, title)` to prevent overlap.
*   **Target Path:** `./sources/backend/src/main/java/org/nlh4j/saas/membershiphub/domain/Enrollment.java`
    *   **Architectural Requirements:**
        *   Entity linking `studentId` and `courseId`; fields `id`, `enrollmentDate`, `status`.
        *   All CRUD operations must use parameterized queries.
*   **Target Path:** `./sources/backend/src/main/java/org/nlh4j/saas/membershiphub/domain/Attendance.java`
    *   **Architectural Requirements:**
        *   Entity for QR check‑in; fields `id`, `studentId`, `courseId`, `attendanceDate`, `qrTokenHash`.
        *   Store SHA‑256 hash of QR token to prevent replay attacks; include `tenantId`.
*   **Target Path:** `./sources/backend/src/main/java/org/nlh4j/saas/membershiphub/domain/Notification.java`
    *   **Architectural Requirements:**
        *   Entity for notifications; fields `id`, `recipientId`, `message`, `sentAt`, `channel` (ZALO, MOBILE_PUSH), `tenantId`.
        *   Implement GDPR‑compliant retention policy; provide delete job.
